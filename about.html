SmartMove

What is SmartMove ?

SmartMove is an application which enables people to move from one place to another. Many others sites as the SNCF's one in France provide this kind of service. However, they sometimes tend to be very difficult to use, and to find...
Here is our innovation: a simple interface where people of every age and abilities can easily search for the information they need.

How to use it?
You type your departure in the first field, and your final destination in the next one.
Tip: You can also put nothing in the first field text and accept that Google determines your current position: in this way, you are searching for a way from where you are to where you want to go.
Then, if you click on the submit button, you can see in the google map the path from the departure to the arrival.
In the tabs above the map, you can choose the way to go: by bus, by car and on foot. On the right side, we can see the information about the CO2, the time it takes to travel, etc.

Technologies involved

Smartmove uses actually a extensive set of techniques and technologies.

  • HTML5: of course, Smartmove relies on the best markup possible: clear, semantic, and valid !
  • CSS3: we tried to implement some CSS3 features like border-radius or text-shadow, to enhance the Smartmove experience
  • HTML5BoilerPlate: initiated by Paul Irish, this open source project aims at providing a solid base for HTML5-cross-browser-compatible websites. HTML5BoilerPlate comes with a lot of techniques and ressources. We didn't use them all of course.
  • Google Map API v3: our main data source.
  • GitHub: a web-based Distributed Control Version System, which enabled a collaborative work on the project.

What does Smartmove does so far ?

The functions that comes with Smartmove today:

  • Visually trace the route from a place to another
  • Automatic geolocation in case the first field in untouched
  • Update of the information according to the mean of transport chosen via the tabs

The limits we encountered

When we let the current location at its default value, once we submit it, the javascript didn't have time to execute as the page reloaded itself.
So the map doesn't display properly the current user's location (even if he accepted to let the browser know it in the first place). We fixed the bug by geolocalising the user when the page loads.
